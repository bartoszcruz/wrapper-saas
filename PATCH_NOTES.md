## üìã Summary

**Changed:** `/api/checkout/route.ts` to use `stripe.subscriptions.update()` for upgrade/downgrade instead of creating new Checkout Sessions.

**Reason:** User preference - upgrades/downgrades should be instant (no Stripe Checkout page), only new subscriptions require Checkout.

---

## üîÑ What Changed

### Before (v2.0):
- **ALL** plan changes ‚Üí Stripe Checkout Session
- User clicks "Subscribe" ‚Üí Redirected to Stripe Checkout ‚Üí Payment/confirmation ‚Üí Webhook ‚Üí Dashboard

### After (v2.1):
- **NEW subscription** ‚Üí Stripe Checkout Session (unchanged)
- **UPGRADE/DOWNGRADE** ‚Üí `stripe.subscriptions.update()` (instant, no Checkout page)
  - User clicks "Subscribe" ‚Üí Backend updates subscription ‚Üí Redirect to dashboard ‚Üí Webhook ‚Üí Polling completes
